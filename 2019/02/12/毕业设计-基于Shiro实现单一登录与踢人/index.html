发表于 2019-02-12 | 分类于 Shiro | | 阅读次数:

前言

接上篇博客，继续完成登录控制，本篇将介绍如何基于Shiro和Redis来实现单一登录与踢出。

项目地址：https://github.com/bigbilii/G-FoodShop

使用场景

对于用户来说，账号以及在某一个设备登录，之后又在另一个设备登录，会把前一个登录踢掉，保证了用户的安全使用。

Shiro实现单一登录和踢人

对于Shiro，提供了自定义拦截器AccessControlFilter来方便开发者自定义拦截所需要的请求。对于我们这个单一登录的需求，可以通过拦截器+session来实现。

配置信息

在配置信息中，配置了之前使用的缓存管理器和会话管理器，他们都是使用redis处理的，还配置了最大会话数为1，踢出前面登录的用户以及踢出后的跳转地址，这些信息也会在之后的代码中体现。

在我们拦截过滤器中配置我们需要自定义拦截器以及需要自定义拦截器拦截的url。
